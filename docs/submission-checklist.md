# Submission Checklist

## Demo Recording

- [ ] Show cat needs care, scan proof, earn points, redeem reward
- [ ] Keep the pet scene centered in the story
- [ ] Show mobile viewport readability
- [ ] Show dark/light/system theme switch from Profile
- [ ] Mention live credential paths only if they were actually tested
- [ ] Label mock/demo-only flows on screen or in narration

## Submission Package

- [ ] README explains the product loop clearly
- [ ] Demo video matches implemented features
- [ ] Screenshots or GIFs are captured from the real app
- [ ] `bun run check`, `bun test`, and `bun run lint` pass
- [ ] Security pre-deploy checklist reviewed
- [ ] No secrets in repo, screenshots, terminal output, or video
- [ ] Known missing `bg_room` and `bg_room_dark` artist assets are not replaced with generated art
