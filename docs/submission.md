# Submission Checklist

Use this before final #hackthekitty submission.

- [ ] README explains Purrward in under two minutes.
- [ ] Demo video recorded and matches implemented features.
- [ ] Screenshots or GIFs captured from the real app.
- [ ] `bun run check`, `bun test`, and `bun run lint` pass.
- [ ] `docs/security.md` checklist completed.
- [ ] Real security scan/report attached if available.
- [ ] Mock/dev-mode scan evidence is labeled as mock, not real.
- [ ] Theme relevance sentence included in the submission text.
- [ ] No accidental personal/team identity in public assets if anonymity is required.
- [ ] No secrets in repo, screenshots, terminal output, or demo video.
